# PRD: 반도체 시료 생산주문관리 시스템 (SampleOrderSystem)

## 1. 배경 및 목적

가상의 반도체 회사 "S-Semi"는 다양한 반도체 시료(Sample)를 생산하여 연구소, 팹리스 업체, 대학 연구실 등에 납품한다. 기존에는 엑셀·메모장으로 주문을 관리하여 다음과 같은 문제가 발생했다.

- 주문 처리 여부를 파악하기 어려움
- 공정 예약 후 완성 시점을 알 수 없음
- 재고가 충분함에도 불필요한 추가 공정이 발생

이를 해결하기 위해 **콘솔 기반**의 "반도체 시료 생산주문관리 시스템"을 개발한다. 담당자가 콘솔에서 직접 명령을 입력하여 시료를 등록하고 주문을 처리한다.

## 2. 역할 및 흐름

| 역할 | 설명 |
|---|---|
| 고객 | 필요한 시료를 이메일 등으로 요청 (시스템 외부) |
| 주문 담당자 | 요청에 맞게 주문서 작성 (시료 예약 등록) |
| 생산 담당자 | 개발 시료 등록, 주문 수신 후 승인/거절, 생산 라인 관리 |

흐름: 고객 요청 → 주문 담당자가 주문 등록(RESERVED) → 생산 담당자가 승인/거절 → (재고 충분 시 CONFIRMED / 재고 부족 시 PRODUCING → 생산 완료 후 CONFIRMED) → 출고 처리(RELEASE)

## 3. 도메인 모델

### 3.1 시료 (Sample)

| 속성 | 설명 |
|---|---|
| 시료 ID | 고유 식별자 (예: S-001) |
| 이름 | 시료명 |
| 평균 생산시간 | 1개 생산에 걸리는 평균 시간 (min/ea) |
| 수율 | 정상 생산 비율 = 정상 시료 수 / 총 생산 시료 수 (예: 0.9) |
| 현재 재고 | 현재 보유 수량 |

### 3.2 주문 (Order)

| 속성 | 설명 |
|---|---|
| 주문번호 | 고유 식별자 (예: ORD-20260416-0043) |
| 시료 ID | 주문 대상 시료 |
| 고객명 | 주문 고객 |
| 주문 수량 | 요청 수량 |
| 상태 | 아래 주문 상태 참조 |

### 3.3 주문 상태 (Order Status)

| 상태 | 의미 |
|---|---|
| RESERVED | 주문 접수 |
| REJECTED | 주문 거절 (정상 흐름 외 상태, 모니터링 대상 제외) |
| PRODUCING | 주문 승인 완료, 재고 부족으로 생산 중 |
| CONFIRMED | 주문 승인 완료, 출고 대기 중 |
| RELEASE | 출고 완료 |

상태 전이:
```
RESERVED --승인(재고 충분)--> CONFIRMED --출고--> RELEASE
RESERVED --승인(재고 부족)--> PRODUCING --생산 완료--> CONFIRMED --출고--> RELEASE
RESERVED --거절--> REJECTED
```

### 3.4 생산 라인 (Production Line)

- 공장에서 시료 하나를 생산하는 설비 흐름
- 하나의 생산 라인은 시료를 하나씩 생산 (단일 라인)
- 주문이 들어온 시료에 대해서만 생산
- 대기 중인 생산 요청은 생산 큐(FIFO)에서 관리

## 4. 기능 명세

### 4.1 메인 메뉴

- 기능별 선택 화면 표시
- 전체 시료 요약 정보(등록 시료 수, 총 재고, 전체 주문 수, 생산라인 대기 건수 등) 표시
- 하위 메뉴: 시료 관리 / 주문(접수) / 주문 승인·거절 / 모니터링 / 출고 처리 / 생산 라인 조회

### 4.2 시료 관리

| 기능 | 설명 |
|---|---|
| 시료 등록 | 새로운 시료를 시스템에 추가. 입력값: 시료 ID, 이름, 평균 생산시간, 수율 |
| 시료 조회 | 등록된 모든 시료 목록과 현재 재고 수량 표시 |
| 시료 검색 | 이름 등 속성으로 특정 시료 검색 |

- 시스템에 등록된 시료만 주문 가능

### 4.3 시료 주문 (예약)

- 고객 요청에 따라 주문 담당자가 주문 생성
- 입력값: 시료 ID, 고객명, 주문 수량
- 등록 시 주문 상태는 RESERVED

### 4.4 주문 승인/거절

| 기능 | 설명 |
|---|---|
| 접수된 주문 목록 | RESERVED 상태 주문 목록 표시 |
| 주문 승인 | 재고 확인 후 자동 분기 처리: <br>• 재고 충분 → 즉시 CONFIRMED 전환 <br>• 재고 부족 → 생산 라인(생산 큐)에 자동 등록, PRODUCING으로 전환 |
| 주문 거절 | 즉시 REJECTED로 전환 |

### 4.5 모니터링

| 기능 | 설명 |
|---|---|
| 주문량 확인 | 상태별(RESERVED / CONFIRMED / PRODUCING / RELEASE) 주문 건수 표시. REJECTED는 무효 주문이므로 제외 |
| 재고량 확인 | 시료별 현재 재고 수량 표시. 주문 대비 재고 수준에 따라 상태 표기: <br>• 여유: 주문 대비 재고 충분 <br>• 부족: 주문 대비 재고 수량 부족 <br>• 고갈: 재고 수량 0 |

### 4.6 생산 라인

- 주문량 대비 부족분을 수율/오차를 고려하여 생산
  - 실 생산량 = `ceil(부족분 / 수율)`
  - 총 생산 시간 = `평균 생산시간 * 실 생산량`
- 생산 완료 시 해당 주문 상태를 PRODUCING → CONFIRMED로 전환
- 생산 현황 표기: 현재 생산 중인 시료의 정보(주문 정보, 진행률 등) 표시
- 대기 주문 확인: 생산 큐에 대기 중인 목록을 FIFO(선입선출) 순서로 표시

### 4.7 출고 처리

- CONFIRMED 상태 주문에 대해서만 출고 가능
- 특정 주문을 선택하여 출고 실행 시 상태를 RELEASE로 전환

## 5. UI 원칙

- 콘솔(CLI) 기반, 메뉴 번호 선택 방식
- 화면 예시는 참고용이며 실제 화면 구성은 자유롭게 설계 가능
- 표기할 정보의 상세 수준(생산 진행률, 예상 완료 시각 등)은 자율적으로 결정
